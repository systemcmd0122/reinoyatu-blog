# 見出し画像機能の実装完了

## 機能概要
新規投稿時に、以下3つの方法で見出し画像を設定できるようになりました：

### 1. 新規アップロード
- ローカルファイルから新しい画像をアップロード
- 自動的に最適化（WebP）して Supabase ストレージに保存
- ハッシュ重複検出で、同じ画像の再アップロードを防止

### 2. ライブラリから選択
- 過去にアップロードした画像を再利用
- ストレージに余分なコピーを作成しない
- 検索機能で該当画像を素早く検索可能

### 3. URLで指定
- 外部 URL から画像を直接参照
- Supabase ストレージを使用しない
- CORS検証済みのサイトの画像を利用可能

## 技術的詳細

### クライアント側（BlogEditor.tsx）
```
新規アップロード: imageFile が設定 → base64Image で送信
ライブラリ選択: imagePreview に既存URL をセット
URL指定: imagePreview に外部URL をセット
```

### サーバー側（actions/blog.ts）
```
if (base64Image)
  → uploadImage() でアップロード
else if (imageUrl !== null && imageUrl !== undefined)
  → その URL を直接使用
else
  → 既存の image_url を保持
```

### ストレージ効率化
- ハッシュ値チェック：同一画像の重複防止
- ライブラリ再利用：既存画像の参照のみ
- 外部URL対応：Supabase ストレージ使用量削減

## 新規コンポーネント
- `ImageLibraryDialog.tsx`: ユーザーのアップロード画像一覧と選択UI
- `ImageURLDialog.tsx`: 外部URL入力とバリデーション

## UIの変更
見出し画像パネル（画像未設定時）に以下3つのボタンを追加：
1. 📤 新規アップロード - ファイル選択
2. 📚 ライブラリから選択 - 過去の画像から選択
3. 🔗 URLで指定 - 外部URLを入力

設定後のホバーUI：
- 変更ボタン（別の画像に切り替え）
- 削除ボタン（画像を削除）
